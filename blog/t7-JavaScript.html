How do Ruby Hashes compare to JS Objects?

July 7, 2015

In my educaiton psych classes in college we talked about knowledge acquisition and how it depends on a person's ability to connect new concepts with information already possessed. In my experience exploring programming, I have found myself thinking about this often regarding my own learning. When introducted to Ruby, I connected it to Algebra. As I am diving into JavaScript, I like connect it to what I have learned so far about Ruby. The more relationships and connections we make, the more easily we learn new information and skills.

In this post, I am going to compare the use of hashes in Ruby to that of objects in JavaScript. In my limited experience thus far, I have found them more similar than not. I know there are nuances and other conceptual discrepancies I have yet to fully understand, but I am going to focus on how you might create a similar structure in both Ruby and JavaScript and how you can apply the syntax rules of each language to accomplish this.

Let's say we would like to create a programming structure that contains key properties of a person's identity. Let's name our structure "identity" and show how you might add these key properties in each of our languages of focus.

Ruby Hash
      1   identity = Hash.new                 #{}
      2   identity[:name] = "Meghan Kline"
      3   identity[:age]  = 25
      4   identity[:hometown] = "Chicago"
      5   p identity

This will print the following to the console:

      {:name=>"Meghan Kline", :age=>25, :hometown=>"Chicago"}
JavaScript Object
      1   var identity = {}                 #{}
      2   identity.name = "Meghan Kline"
      3   identity.age  = 25
      4   identity.hometown = "Chicago"
      5   console.log(identity)

This will print the following to the console:

      { name: 'Meghan Kline', age: 25, hometown: 'Chicago' }

As you can see, the creation of a Ruby hash and a JS object are quite similar with a few syntactic differences. In the hash the keys are symbols in this case and the values are either strings or numbers. In the JS object the properties are specified and the values are again either strings or numbers. We can retrieve the value of any given key/property by using appropriate syntax.

Ruby Hash
      6   identity[:name]   # "Meghan Kline"
      7   identity.key(25)  # :age
JS Object
      6   identity.name     // 'Meghan Kline'
      7   identity["name"]  // 'Meghan Kline'
      // There is not an easy built in method for retrieving a property 
      // for a given value like using hash.key(value) in Ruby.

In general, there are many more built in methods in Ruby that allow you to manipulate hashes and other Ruby objects in many ways. JavaScript will allow for more variabiltiy than Ruby, but does not present errors as easily, which can often lead to a difficult debugging process when your code does not return what you expect. However, there are ways to modify values and iterate over objects in JS similar to Ruby. The problem is that it's more difficult to modify or access the "key" equivalent in JS than in Ruby.

Let's modify some of the values in each of our structures. Let's say this Meghan character gets married and hyphenates her name to Kline-Tuls. Also, she's aged since then to 28. And she has since moved to sunny San Diego.

Ruby Hash
      8   identity[:name] = "Meghan Kline-Tuls"
      9   identity[:age]  = 28
      10  identity[:hometown] = "San Diego"
JS Object
      8   identity.name = "Meghan Kline-Tuls"
      9   identity["age"]  = 28
      10  identity.hometown = "San Diego"

Another feature that is slightly different between the structures is how to iterate over each.

Ruby Hashes
      identity.each do |property, value|
        puts "The person's #{property} is #{value}."
      end
JS Object
      for (var property in identity) {
  console.log("The person's " + property + " is " +  identity[property] + ".");
}

Both codes will print the same information to the console:

      The person's name is Meghan Kline-Tuls.
      The person's age is 28.
      The person's hometown is San Diego.
